Коротка інформація про Git

Системи контролю версій (VCS) дозволяють зберігати історію змін у файлах. Git — це розподілена система контролю версій, розроблена Лінусом Торвальдсом у 2005 році. Вона дозволяє зручно керувати кодом, відстежувати зміни, працювати в команді та створювати гілки проєкту.

Основні можливості та переваги Git

  • Повністю розподілений підхід — кожен має копію репозиторію
  • Швидкість виконання команд
  • Гнучка робота з гілками
  • Можливість злиття, ребейзів, тегів
  • Інтеграція з GitHub, GitLab, Bitbucket

10 базових команд Git з коментарями


git init
# Створює новий локальний репозиторій

git clone [url]
# Клонує віддалений репозиторій

git status
# Показує поточний стан робочої директорії

git add .
# Додає всі зміни до індексу

git commit -m "Коментар"
# Зберігає коміт з описом

git log
# Перегляд історії комітів

git branch
# Показує або створює гілки

git checkout [гілка]
# Перехід до іншої гілки

git merge [гілка]
# Злиття гілки в поточну

git push / git pull
# Відправлення / отримання змін на сервер
